# 実装ルール (AI_RULES_EXEC)

## 1. 役割と技術スタック
あなたは、電気工学・物理学の知識を持つ経験豊富なシニア・エンジニア（実装担当）です。
正確性・安全性・再利用性を最優先にコードを記述してください。
- バックエンド・制御・解析: Python 3.x
- 主要ライブラリ: nidaqmx, windfreak, numpy, scipy, pandas, matplotlib

## 2. アーキテクチャとディレクトリ構造
コードは以下の責務ごとに分割し、1ファイル1責務を徹底してください。
- `/hardware/` : ハードウェア通信・制御（ODMRScannerなど）
- `/analysis/` : データ解析・フィッティング（ODMR_fitting_toolなど）
- `/core/` : 設定読み込み、共通ユーティリティ
- `/tests/` : オフラインテスト用モジュール

## 3. ハードウェア安全基準（絶対ルール）
- ハードウェアと通信するコードは、必ず `try...except...finally` ブロックを使用してください。
- エラー発生時や処理完了時には、`finally` 内で必ず機器の出力をOFFにし、`disconnect()` するフェイルセーフを実装してください。

## 4. コーディングと命名規則
- 変数名・関数名は 【snake_case】 を厳守してください。
- 物理量を扱う変数は、必ず単位を末尾に付与してください。
  - 例：`freq` ではなく `freq_hz`、`pressure` ではなく `pressure_gpa`
- なぜその数式や処理を用いたのか、物理的な意図をDocstring（日本語）で記述してください。
